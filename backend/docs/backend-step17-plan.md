# STEP 17 계획 — Transcript (트랙·세그먼트)

## 1. 단계 목표

- 공용 **Video** 기준 **자막/스크립트**를 **TranscriptTrack + TranscriptSegment** 로 모델링하고, 저장·조회·트랙 선택·동기화(스텁) API를 제공한다.

## 2. 이번 단계에서 해결할 문제

- 영상 상세 학습 UX의 기반 데이터(시간轴 텍스트)를 DB에 두고, 노트/하이라이트와 **도메인 분리**한다.
- 자막이 **없는 것이 정상**인 경우를 API 계약으로 명확히 한다.

## 3. 설계 대상

- 엔티티: `TranscriptTrack`(video, languageCode, autoGenerated, source, selected), `TranscriptSegment`(track, lineIndex, start/end seconds, text).
- 유니크: `(video_id, language_code, is_auto_generated)`; 세그먼트 `(transcript_track_id, line_index)`.
- API base: `/api/v1/videos/{videoId}/transcript` (`videoId` = `Video.id`).
- 접근: JWT 사용자가 해당 `video_id`의 **UserVideo를 보유**할 때만 허용(403).

## 4. 주요 결정 사항

- Entity 직접 응답 금지, DTO(`TranscriptViewResponse` 등)로 **tracksAvailable / hasSelectedTrack / segments** 구조 고정.
- 동기화: 트랙별 **세그먼트 전면 교체**; 업스트림 빈 결과는 **200 + upstreamEmpty**.
- Infra: `YoutubeTranscriptPort` + **Stub**(실 API 미연동).
- 로그: `videoId`, `trackId`, `languageCode`, `segmentCount` — **본문 텍스트 대량 로깅 금지**.

## 5. 생성/수정 예정 파일

- `domain/transcript/*`, `infra/youtube/transcript/*`, `ErrorCode`, `GlobalExceptionHandler`, 문서.

## 6. 구현 범위

- Swagger, 공통 예외, `VIDEO_NOT_FOUND` / `TRANSCRIPT_*` / `COMMON_CONFLICT`.

## 7. 제외 범위

- 스크립트 검색·FTS·AI 요약, 실제 YouTube captions 다운로드 완성 구현.

## 8. 다음 단계 연결 포인트

- 배치/`SyncJob`로 주기적 sync, UserVideo import 후 비동기 자막 수집, `YoutubeTranscriptRestAdapter`, DB 전문 검색.
