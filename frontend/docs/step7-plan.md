# step7-plan — 나중에 보기를 학습 큐로 재구성

## 1. 단계 목표

- **`/watch-later` 페이지**를 “쌓인 목록”이 아니라 **오늘·이후 학습 계획을 세우는 공간**으로 구현한다.
- **학습용/비학습용** 의도, **우선순위**, **오늘 볼 큐**, **오래된 항목 표시**, **일괄 정리**, **컬렉션 이동**을 한 화면에서 다룬다.

---

## 2. 이번 단계에서 해결할 사용자 문제

| 문제 | 대응 |
|------|------|
| 나중에 보기만 쌓여 부담 | 상단 **오늘 볼 영상**으로 “지금 할 양”만 분리, 부드러운 카피 |
| 학습·휴식 영상이 섞임 | **학습용 / 비학습용** 배지·필터 |
| 무엇부터 볼지 모름 | **우선순위** + 큐 **순서 조정**(mock) |
| 오래 묵은 영상 방치 | **추가일 기준 오래됨** 배지(일수 임계값) |
| 하나씩만 정리 가능 | **선택 + 일괄 액션** 바 |
| 학습 자산 구조로 넘기고 싶음 | **컬렉션 이동**(기존 `VIDEO_COLLECTIONS` id) |

---

## 3. 대상 페이지 / 컴포넌트

- **페이지**: `WatchLaterPage`.
- **피처** `src/features/watch-later-management/`: `filterWatchLater`, `WatchLaterToolbar`, `TodayQueueStrip`, `WatchLaterListRow`, `BulkActionBar`, `watch-later.css`.
- **재사용**: `PageHeader`, `FilterBar`, `Button`, `EmptyState`.

---

## 4. 화면 구성 요소

1. **PageHeader** — “학습 계획” 톤, 학습 자산으로 이동 CTA.
2. **오늘 볼 영상** — 큐에 넣은 항목만 순서대로, 위/아래·큐 제거.
3. **툴바** — 검색, 정렬(계획 순·우선순위·담은 날짜 등).
4. **필터** — 의도 전체 / 학습용 / 비학습용.
5. **목록 행** — 체크박스, 제목·채널·길이, 의도·우선순위·오래됨, 오늘 큐 토글, 컬렉션·학습 자산 이동(선택 id 있을 때).
6. **일괄 액션 바** — 선택 시 하단 고정: 큐 넣기/빼기, 의도·우선순위, 컬렉션, 제거.

---

## 5. 상태 및 데이터 구조

- `WatchLaterEntry`, `WatchLaterIntent`, `WatchLaterFilterState` (`shared/types/watch-later.ts`).
- Mock: `WATCH_LATER_MOCK`, 상수 `WATCH_LATER_STALE_DAYS`.
- 페이지 로컬 상태: 목록 배열, `Set` 기반 선택 id.

---

## 6. UX 결정 사항

- **압박 완화**: 헤더·빈 상태에서 “다 걸러내야 한다”가 아니라 **오늘 1~3개만 고르면 된다**는 메시지.
- **계획 언어**: “정리”보다 **오늘 큐 편성**, **의도(학습/비학습)** 라벨.
- **오래됨**: 경고색 과다 사용 금지 — **muted 배지 + ‘다시 보기’ 제안** 수준.
