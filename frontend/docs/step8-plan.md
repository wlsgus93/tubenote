# step8-plan — 메모·하이라이트 아카이브(학습 흔적 허브)

## 1. 단계 목표

- **`/notes` 페이지**에서 영상별로 흩어진 **타임라인 메모**와 **하이라이트**를 한곳에 모아 **학습 자산처럼** 탐색한다.
- **태그·복습 필요·종류(메모/하이라이트)** 필터와 **최근 작성순 / 중요도순** 정렬, **카드·리스트** 보기를 제공한다.
- 각 항목에서 **영상 상세로 이동** 및 **해당 시점(`?t=초`)으로 점프**가 가능하다.

---

## 2. 이번 단계에서 해결할 사용자 문제

| 문제 | 대응 |
|------|------|
| 메모가 영상 안에만 있어 전체 맥락이 안 보임 | 통합 목록 + 영상·채널·시점 메타 |
| 복습할 것만 보고 싶음 | **복습 필요만** 필터 |
| 주제별로 모아보고 싶음 | **태그** 칩 필터 |
| 기록 vs 인용 구분 | **메모 / 하이라이트** 필터·배지 |
| 다시 그 장면으로 가기 번거로움 | **`/videos/:id?t=초`** + 상세에서 시크 |

---

## 3. 대상 페이지 / 컴포넌트

- **페이지**: `NotesPage`.
- **피처** `src/features/note-archive-management/`: `filterNoteArchive`, `NoteArchiveToolbar`, `NoteArchiveCard`, `NoteArchiveListRow`, `note-archive.css`.
- **재사용**: `PageHeader`, `FilterBar`, `Button`, `EmptyState`.
- **연동**: `VideoDetailPage`에 `t` 쿼리 처리(메모 탭 포커스 + 재생 시각).

---

## 4. 화면 구성 요소

1. **PageHeader** — “학습 흔적” 톤, 학습 자산 링크.
2. **툴바** — 검색, 정렬, 그리드/리스트 전환.
3. **필터** — 종류(전체·메모·하이라이트), 복습 필요, 태그.
4. **결과 영역** — 카드 또는 리스트 행: 흔적 유형, 본문, 태그·중요도·복습, CTA **이 시점으로 학습 화면 열기**.

## 6. UX 결정 사항

- **학습 흔적** 언어: eyebrow·부제에 “기록”보다 맥락(영상·시점·의도) 강조.
- **복습**는 뱃지 + 필터로 빠른 진입.
- 정보 밀도는 카드(그리드) / 한 줄 요약(리스트)로 사용자 선택.
